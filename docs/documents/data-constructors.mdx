---
title: "Data constructors"
---

To pass a specific complex data type, use provided data constructors when inserting or querying your documents.

TopK provides constructors for the following complex types:

### f32_vector()

To pass a float32 vector, use the `f32_vector()` helper function:

<CodeGroup>

```python Python
from topk_sdk.data import f32_vector

f32_vector([0.12, 0.67, 0.82, 0.53])
```


```typescript Javascript
import { f32Vector } from "topk-js/data";

f32Vector([0.12, 0.67, 0.82, 0.53])
```

</CodeGroup>

<Info>
  When passing an array of numbers, it is by default considered a list of float32 values.
</Info>

### f32_list()

To pass a list of 32-bit floating-point numbers, use the `f32_list()` helper function:


<CodeGroup>

```python Python
from topk_sdk.data import f32_list

f32_list([0.12, 0.67, 0.82, 0.53])
```


```typescript Javascript
import { f32List } from "topk-js/data";

f32List([0.12, 0.67, 0.82, 0.53])
```

</CodeGroup>

<Info>
  When passing an empty list or a list of numbers such as `my_list = [0.12, 0.67, 0.82]` or `my_list = []`, it is by default considered a list of float32 values.
</Info>

### string_list()

To pass a list of strings, use the `string_list()` helper function:

<CodeGroup>

```python Python
from topk_sdk.data import string_list

string_list(["foo", "bar", "baz"])
```


```typescript Javascript

```

</CodeGroup>

<Tip>This is specifically useful when you need to pass an empty list of strings `string_list([])` as lists are assumed to be of type `float32` by default.</Tip>
